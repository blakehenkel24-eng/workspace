# SlideTheory Idea Repository

A structured system for capturing and organizing business ideas, content ideas, and feature requests.

## Repository Structure

```
ideas/
├── README.md              # This file - overview and usage guide
├── business/              # Business & monetization ideas
│   ├── README.md
│   ├── revenue-streams.md
│   └── partnerships.md
├── content/               # Content & marketing ideas
│   ├── README.md
│   ├── article-ideas.md
│   ├── linkedin-posts.md
│   └── lead-magnets.md
├── features/              # Product feature requests
│   ├── README.md
│   ├── core-features.md
│   ├── integrations.md
│   └── templates.md
└── archive/               # Implemented/rejected ideas
    └── README.md
```

## Quick Capture Template

When you have a new idea, use this format:

```markdown
### Idea: [Brief Title]

**Category:** business | content | feature  
**Priority:** high | medium | low  
**Status:** new | researching | planned | archived  
**Tags:** #tag1 #tag2

**Description:**
[1-2 sentence description]

**Why it matters:**
[Business case or user benefit]

**Effort estimate:**
[s/m/l or hours]

**Notes:**
[Any additional context, links, or references]
```

## Status Definitions

| Status | Meaning |
|--------|---------|
| new | Just captured, needs evaluation |
| researching | Gathering info, validating |
| planned | Approved, queued for implementation |
| in-progress | Currently being worked on |
| implemented | Done and live |
| archived | Rejected or deprecated |

## Tag Reference

**Business Tags:**
- `#revenue` - Direct monetization
- `#growth` - User acquisition
- `#retention` - Keeping users engaged
- `#partnership` - Collaboration opportunities

**Content Tags:**
- `#thought-leadership` - Establishing expertise
- `#seo` - Search optimization
- `#viral` - High share potential
- `#evergreen` - Long-term value

**Feature Tags:**
- `#ai` - AI/ML functionality
- `#ux` - User experience
- `#export` - Output formats
- `#integration` - Third-party connections
- `#enterprise` - B2B features
